# Publishing to DefinitelyTyped

This document describes the process for updating the official [`@types/fhir`](https://www.npmjs.com/package/@types/fhir) package on [DefinitelyTyped](https://github.com/DefinitelyTyped/DefinitelyTyped).

## Overview

This repository generates the source files for `@types/fhir`. The generated `.d.ts` files and test files are **not committed here** — they are produced on demand and copied into a fork of DefinitelyTyped before opening a PR.

## Prerequisites

- [Bun](https://bun.sh) ≥ 1.0 installed
- A fork of [DefinitelyTyped](https://github.com/DefinitelyTyped/DefinitelyTyped) cloned locally

## Step 1: Generate

Run the generator to produce the declaration files and DefinitelyTyped test files:

```bash
bun run generate:types
```

This reads `packages/types-fhir/generate.config.ts` and writes:

- `packages/types-fhir/index.d.ts`
- `packages/types-fhir/r2.d.ts` through `r5.d.ts`
- `packages/types-fhir/test/r2-tests.ts` through `r5-tests.ts`

## Step 2: Typecheck

Verify the generated output typechecks using the same tsconfig that DefinitelyTyped uses:

```bash
cd packages/types-fhir && bun run typecheck
```

Fix any errors before proceeding.

## Step 3: Copy to DefinitelyTyped

Copy the generated files into your DefinitelyTyped fork:

```bash
DT_FORK=/path/to/DefinitelyTyped

cp packages/types-fhir/*.d.ts        $DT_FORK/types/fhir/
cp packages/types-fhir/tsconfig.json $DT_FORK/types/fhir/
cp -r packages/types-fhir/test/      $DT_FORK/types/fhir/test/
```

## Step 3b: Fix package.json

dtslint requires that `types/fhir/package.json` does **not** contain a `name` field. The DT package.json should look like:

```json
{
  "private": true,
  "dependencies": {},
  "devDependencies": {}
}
```

If the file has a `name` (or other disallowed fields), remove them before running dtslint.

## Step 4: Run dtslint

From inside the DefinitelyTyped fork, run the official linter:

```bash
cd $DT_FORK/types/fhir
npx dtslint .
```

Resolve any dtslint errors. Some tests may require adjustment to match dtslint's expectations for assertion syntax.

## Known exceptions

Certain FHIR examples from the official packages fail type inference or are intentionally skipped:

- **`EXCEPTIONS`** — types that are tested but with adjusted expectations (see `packages/fhir-ts-codegen/src/generate/tests.ts`)
- **`SKIP_TYPES`** — types excluded from test generation entirely (see `packages/fhir-ts-codegen/src/generate/tests.ts`)

These maps are maintained in the generator source and are applied during `generate:types`.

## Step 5: Open a PR

Follow the [DefinitelyTyped contribution guide](https://github.com/DefinitelyTyped/DefinitelyTyped/blob/master/README.md#make-a-pull-request).

In the PR description, mention that the types are generated by [`@rh/fhir-ts-codegen`](https://github.com/reason-healthcare/fhir-types/packages/fhir-ts-codegen) from official FHIR NPM packages (`hl7.fhir.r*.core`), not hand-authored. This helps reviewers understand the scope of changes.
